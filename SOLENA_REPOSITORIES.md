# Solena Repository Map

This file records the two-repository setup so the project stays organized over time.

## Public repository

Purpose:

- showcase the project publicly
- provide a clean GUI
- explain the workflow at a high level
- avoid sensitive dialogue history

Main contents:

- `index.html`
- `styles.css`
- `app.js`
- `README.md`

## Private repository

Purpose:

- keep raw dialogues
- keep internal notes and reasoning
- keep governance rules
- keep versioning and lab material

Location:

- `private-core/`

## Rule

- public is for visibility
- private is for the real working memory
- sensitive material stays private unless explicitly cleaned for publication
